# proto-to-prd.skill
从原型工程（Vue/HTML代码文件）逆向生成结构化需求说明文档（PRD）。当用户提到解读原型、反向生成PRD、代码转需求文档、原型工程需求分析、结构化需求说明、原型评审、代码生成需求等意图时使用此技能。支持两阶段流程：阶段1以PM视角评审原型代码生成需求确认清单（含页面结构、功能说明、关联代码文件、置信度、产品决策、PRD建议），用户确认后进入阶段2生成完整PRD（含文档背景、目标用户、核心业务流程、功能清单、功能说明含事件流/需求规则/字段说明/数据需求、接口需求、安全需求）及交互式需求查看Vue组件，支持增量更新和PRD与代码双向追溯。

## 使用指南

### 快速开始
1. **准备原型工程**：确保你的原型工程包含 Vue（.vue）或 HTML（.html）文件
2. **触发技能**：告诉我"解读这个原型"或"反向生成PRD"，并提供 zip 包或目录路径
3. **确认需求清单**：审阅阶段1生成的需求确认清单，标注确认/修改/遗漏
4. **获取PRD**：确认后自动生成结构化PRD和交互视图组件
5. **集成到工程**：按 install.sh 说明将交互视图组件嵌入原型工程

### 典型工作流组合
**流程 A：原型 → PRD → 代码补全**
```
原型工程 → 本技能(逆向PRD) → Trae SOLO模式(按PRD补全代码) → 可运行应用
```

**流程 B：遗留系统 → 文档化 → 重构**
```
遗留代码库 → 本技能(逆向PRD) → Codex $recipe-review(合规校验) → Codex $recipe-build(重构实施)
```

**流程 C：设计稿 → 原型 → PRD → 企业交付**
```
Figma设计稿 → CodeBuddy(生成原型) → 本技能(逆向PRD) → TAPD(需求入库) → 企业开发流程
```

## 输入规范
用户调用时需提供：
- **工程来源**：zip 包上传 或 目录路径
- **（可选）L4增强**：是否启用框架层深度解析，默认关闭
- **（可选）输出目录**：默认为 outputs/

## 输出规范
详见步骤9的交付产物清单。所有输出文件使用 UTF-8 编码，Markdown 文件使用标准 GitHub Flavored Markdown 格式。
